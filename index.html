1.11 HTML Block & Inline Elements

In this lesson, I learned about the difference between block and inline elements in HTML. Understanding how they behave helps in structuring and styling web pages correctly.

Block Elements

Block elements take up the entire width of the page and always start on a new line. Common examples include headings, paragraphs, and divs. 

<h1>This is a heading</h1>
      <p>This is a paragraph</p>
      <div>This is a block div</div>

Each of these starts on a new line and stretches across the page, creating a clear structure in the layout.

Inline Elements

Inline elements only take up as much space as needed and sit within block elements. They‚Äôre useful for styling parts of text or adding links inside a paragraph. 


      <span>This is a span</span>
      <a href="#">This is a link</a>
      <strong>This is bold</strong>

These stay within the same line of text ‚Äî perfect for highlighting or linking content without breaking the flow.

Visual Difference

When displayed on a webpage, block elements stack one below the other, while inline elements appear next to each other on the same line.

What I Learned üß†

I learned that block elements structure the layout by creating sections, while inline elements style or emphasize content within those sections.Knowing when to use each type is essential for clean and organized web design.

1.13 CSS Selectors

In this lesson, I learned how to use CSS selectors to style specific parts of a webpage. Selectors help target elements so each one can have a unique look.

What are CSS Selectors?

Selectors are like giving instructions to the browser about which elements to style. Without them, every element would look the same!

Tag Selectors

Tag selectors style all elements of a particular type. Example: Making all headings green and all paragraphs larger. 

h1 {
        color: green;
      }

      p {
        font-size: 18px;
      }

Class Selectors

Class selectors style elements with a specific class attribute. This allows multiple elements to share the same style. 

.highlight {
        background-color: yellow;
      }

ID Selectors

ID selectors style a single element with a specific id attribute. This is useful for unique elements that need distinct styling. 

#main-title {
        font-size: 24px;
        font-weight: bold;
      }

Pseudo-Class Selectors

Pseudo-class selectors style elements based on their state or position. For example, changing link colors when hovered over. 

a:hover {
        color: red;
      }

What I Learned üß†

    Tag selectors are for all elements of a type.
  • Class selectors are for groups of elements with a common role.
  • ID selectors are for a unique element.
  • Pseudo-classes are for states or interactions.
  • CSS selectors give me precise control, like a director controlling each actor on stage. Knowing how to use them lets me make web pages look exactly the way I want.

2.1 Font & Text Properties in CSS

In this lesson, I learned how CSS allows me to style text and control its appearance on a webpage. Typography is important for readability and making the page visually appealing, so understanding font and text properties is essential.

  • Apply CSS properties to adjust text layout (alignment, spacing, decoration).
  • Choose appropriate font families for different contexts (serif, sans-serif, monospace).
  • Understand the impact of typography on readability and user experience.
  • Combine multiple text and font properties to create styled text elements.

Font Families

I learned that font families are like outfits for text. Serif fonts have small "feet" (e.g., Times New Roman, Georgia) and look formal. Sans-serif fonts have no feet (e.g., Arial, Helvetica) and look clean and modern. Monospace fonts (e.g., Courier New) make each letter the same width and are ideal for code.

The font-family property lets me specify which font to use and provide fallback options. For example:

font-family: "Courier New", Courier, monospace;

Font Size, Weight, and Style

  • font-size ‚Äì Controls the size of text. Can use px, em, or rem.
  • font-weight ‚Äì Controls thickness or boldness (values: normal, bold, or 100‚Äì900).
  • font-style ‚Äì Controls slanting: normal, italic, or oblique.

Text Properties

  • text-align ‚Äì Align text: left, right, center, or justify.
  • text-decoration ‚Äì Add/remove lines: none, underline, overline, line-through.
  • line-height ‚Äì Space between lines (e.g., 1.5 for readability).
  • text-transform ‚Äì Change capitalization: uppercase, lowercase, capitalize, none.

2.2 CSS Inheritance

In this lesson, I learned about CSS inheritance, which allows child elements to inherit certain property values from their parent elements. Understanding inheritance helps me write cleaner and more efficient CSS.

What I Learned

Inheritance in CSS works similarly to how children inherit traits from their parents. For example, if I set a property on a parent element, its child elements may inherit that property automatically.

Example

HTML:

<p>I am a <strong>KALVIAN</strong></p>

CSS:

p {
        color: red;
        }
        strong {
        font-size: 16px;
       }

Result: The entire sentence appears red because the <strong> tag inherits the color: red; property from its parent <p> element.

Important Notes

  • Not all CSS properties are inherited. For example, border is not inherited by default.
  • Commonly inherited properties include: color, direction, font-family, font-size, font-style, font-weight, line-height, text-align, and several others.
  • Understanding which properties inherit helps avoid unintended styles on child elements.

Practice

I practiced CSS inheritance using a web playground like Stackblitz to see how properties propagate from parent to child elements and which properties are not inherited.

2.3 CSS Positioning

In this lesson, I learned how to control the exact placement of elements on a webpage using CSS positioning. Instead of letting the browser decide the layout, I can now place elements exactly where I want.

CSS Positioning Types

1. Static (default)

By default, every element has position: static;. Elements appear in the order they are written in HTML, from top to bottom. You cannot move them using top, bottom, left, or right.

2. Relative

With position: relative;, elements stay in their normal flow but can be nudged using top, right, bottom, and left.

Relative Box

3. Absolute

Absolute positioning removes the element from the normal flow. It is positioned relative to the nearest parent with a position other than static. If no parent is positioned, it will use the browser window as reference.

Container
Absolute Box

4. Fixed

Fixed elements stay in place relative to the browser viewport. They do not move when the page scrolls.

Fixed Box

5. Sticky

Sticky elements act like relative until they reach a scroll threshold, then they stick like fixed.
